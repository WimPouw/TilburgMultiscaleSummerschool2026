Audio Masking &
Multi-Modal Considerations

Voice is biometric data. Masking the video is not enough.

Babajide Owoyele · Radboud University / SYNAPSIS

Why isn't masking the video enough?

  • Voice is biometric — speaker identification works with < 5 seconds of audio
  • Existing tools — Microsoft Azure, Google Speech, AWS Transcribe all have speaker ID APIs
  • GDPR, Art. 9 — voice recordings are special-category biometric data
  • Ethics boards increasingly require both video and audio anonymisation
A masked video with an unmodified audio track is not anonymised — the audio alone re-identifies the speaker.
Even removing the audio track doesn't fully anonymise — lip movements and silent gestures can sometimes decode speech.

Three de-identification strategies

1. Pitch shifting

Raise or lower F0 by ±4 semitones. Fast, reversible. Prosody largely preserved.


Privacy: low–med

2. Noise + formant

Add calibrated noise, shift formants via resample. Confuses speaker ID models.


Privacy: medium

3. Speech → Text → Speech

Transcribe with Whisper, re-synthesise with TTS. Original voice is gone entirely.


Privacy: high
Open notebooks/04_audio_deidentification.ipynb to run all three methods on your own audio.

The privacy–utility trade-off in audio

Method Speaker ID blocked? Pitch preserved? Prosody preserved? Intelligible?
Pitch shift Partial No (shifted) Yes Yes
Noise + formant Mostly Approx. Partial Yes
S → T → S Yes No No Yes (content)
Strip audio Yes N/A N/A No audio

Multi-speaker sessions: diarise first

The Envision triadic sessions have 3 speakers. Apply a different anonymisation per speaker so they remain distinguishable.


# pyannote speaker diarization → per-speaker pitch shifts
pipeline = Pipeline.from_pretrained("pyannote/speaker-diarization-3.1")
diarization = pipeline("group_07_cam_p1.wav")

speaker_shifts = {}           # e.g. SPEAKER_00 → +4st, SPEAKER_01 → -4st
shift_pool = [+4, -4, +6]

for turn, _, speaker in diarization.itertracks(yield_label=True):
    if speaker not in speaker_shifts:
        speaker_shifts[speaker] = shift_pool[len(speaker_shifts)]
    # apply pitch_shift() to this segment of audio

Connection to Tuesday acoustic analysis

What Tuesday measured

  • Fundamental frequency (F0) — pitch
  • Intensity / energy
  • Formant frequencies (F1, F2)
  • Speech rate, pauses

What de-id destroys

  • Pitch shift — F0 values are invalid
  • Noise injection — intensity and formants distorted
  • S → T → S — all prosodic detail lost
Best practice: extract and archive all acoustic features you need before de-identifying the audio.

Quick reference: audio operations with ffmpeg 


# Extract audio track as WAV (for analysis or de-id)
ffmpeg -i session.mp4 -vn -acodec pcm_s16le audio.wav

# Strip audio entirely (for open-tier archive)
ffmpeg -i masked.mp4 -an -c:v copy masked_silent.mp4

# Replace audio track with de-identified version
ffmpeg -i masked_video.mp4 -i deid_audio.wav \
  -map 0:v -map 1:a -c:v copy -c:a aac \
  masked_with_deid_audio.mp4

# Apply pitch shift via ffmpeg's rubberband filter (no Python needed)
ffmpeg -i audio.wav -af "rubberband=pitch=1.26" shifted.wav

pitch 1.26 ≈ +4 semitones   pitch 0.79 ≈ −4 semitones
